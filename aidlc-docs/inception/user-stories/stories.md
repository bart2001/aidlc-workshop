# 테이블오더 서비스 - 사용자 스토리

## 스토리 구성 정보

- **분류 방식**: 기능 기반 (Feature-Based)
- **수용 기준 형식**: Given/When/Then (표준, 5~8개)
- **우선순위**: MoSCoW (Must/Should/Could/Won't)
- **페르소나**: 고객(김민수), 관리자(박서연)

---

## 고객용 스토리 (Customer Stories)

### US-C01: 테이블 태블릿 자동 로그인

**스토리**: 매장 고객으로서, 테이블에 앉으면 별도 로그인 없이 바로 메뉴를 볼 수 있기를 원한다, 대기 시간 없이 즉시 주문하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 태블릿에 로그인 정보가 저장되어 있을 때 | 태블릿 브라우저를 열면 | 자동으로 로그인되어 메뉴 화면이 표시된다 |
| 2 | 태블릿에 로그인 정보가 없을 때 | 브라우저를 열면 | 매장 식별자, 테이블 번호, 비밀번호 입력 화면이 표시된다 |
| 3 | 관리자가 초기 설정 정보를 입력했을 때 | 로그인에 성공하면 | 로그인 정보가 로컬에 저장되고 메뉴 화면으로 이동한다 |
| 4 | 잘못된 인증 정보를 입력했을 때 | 로그인을 시도하면 | 에러 메시지가 표시되고 재입력을 요청한다 |
| 5 | 세션이 만료되었을 때 | 페이지에 접근하면 | 저장된 정보로 자동 재로그인을 시도한다 |

---

### US-C02: 메뉴 조회 및 탐색

**스토리**: 매장 고객으로서, 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하고 싶다, 원하는 음식을 빠르게 찾기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 로그인된 상태에서 | 메뉴 화면에 진입하면 | 카테고리 목록과 해당 카테고리의 메뉴가 카드 형태로 표시된다 |
| 2 | 메뉴 목록이 표시된 상태에서 | 다른 카테고리 탭을 선택하면 | 해당 카테고리의 메뉴로 빠르게 전환된다 |
| 3 | 메뉴 카드가 표시된 상태에서 | 메뉴 카드를 확인하면 | 메뉴명, 가격, 설명, 이미지가 표시된다 |
| 4 | 메뉴 화면에서 | 터치 조작을 할 때 | 모든 버튼/터치 영역이 최소 44x44px 이상이다 |
| 5 | 매장에 등록된 메뉴가 없을 때 | 메뉴 화면에 진입하면 | "등록된 메뉴가 없습니다" 안내 메시지가 표시된다 |

---

### US-C03: 장바구니 관리

**스토리**: 매장 고객으로서, 주문 전에 선택한 메뉴를 장바구니에서 확인하고 수정하고 싶다, 정확한 주문을 하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 메뉴 화면에서 | 메뉴 항목의 추가 버튼을 누르면 | 해당 메뉴가 장바구니에 추가되고 수량이 1로 설정된다 |
| 2 | 장바구니에 메뉴가 있을 때 | 같은 메뉴를 다시 추가하면 | 해당 메뉴의 수량이 1 증가한다 |
| 3 | 장바구니에 메뉴가 있을 때 | 수량 감소 버튼을 누르면 | 수량이 1 감소하고, 0이 되면 항목이 제거된다 |
| 4 | 장바구니에 메뉴가 있을 때 | 장바구니를 확인하면 | 각 메뉴의 수량, 단가, 소계와 총 금액이 표시된다 |
| 5 | 장바구니에 메뉴가 있을 때 | 장바구니 비우기를 누르면 | 모든 항목이 제거되고 총 금액이 0원이 된다 |
| 6 | 장바구니에 메뉴가 있을 때 | 페이지를 새로고침하면 | 장바구니 내용이 유지된다 (로컬 저장) |
| 7 | 장바구니가 비어있을 때 | 장바구니를 확인하면 | "장바구니가 비어있습니다" 안내와 메뉴로 이동 버튼이 표시된다 |

---

### US-C04: 주문 생성

**스토리**: 매장 고객으로서, 장바구니의 메뉴를 주문으로 확정하고 싶다, 음식을 받기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 장바구니에 메뉴가 있을 때 | 주문하기 버튼을 누르면 | 주문 내역(메뉴, 수량, 금액) 최종 확인 화면이 표시된다 |
| 2 | 주문 확인 화면에서 | 주문 확정 버튼을 누르면 | 서버에 주문이 전송된다 |
| 3 | 주문이 성공했을 때 | 결과 화면이 표시되면 | 주문 번호가 표시되고 5초 후 메뉴 화면으로 자동 리다이렉트된다 |
| 4 | 주문이 성공했을 때 | 메뉴 화면으로 돌아오면 | 장바구니가 자동으로 비워져 있다 |
| 5 | 주문이 실패했을 때 | 결과 화면이 표시되면 | 에러 메시지가 표시되고 장바구니 내용은 유지된다 |
| 6 | 주문 정보가 서버에 전송될 때 | 서버가 수신하면 | 매장 ID, 테이블 ID, 메뉴 목록, 총 금액, 세션 ID가 포함되어 있다 |

---

### US-C05: 주문 내역 조회

**스토리**: 매장 고객으로서, 현재 테이블에서 주문한 내역과 상태를 확인하고 싶다, 주문 진행 상황을 파악하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 주문이 존재할 때 | 주문 내역 화면에 진입하면 | 현재 세션의 주문만 시간 순으로 표시된다 |
| 2 | 주문 목록이 표시된 상태에서 | 각 주문을 확인하면 | 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다 |
| 3 | 관리자가 주문 상태를 변경했을 때 | 고객 화면에서 | SSE를 통해 실시간으로 상태가 업데이트된다 |
| 4 | 테이블 세션이 종료(이용 완료)된 후 | 새 고객이 주문 내역을 확인하면 | 이전 세션의 주문은 표시되지 않는다 |
| 5 | 현재 세션에 주문이 없을 때 | 주문 내역 화면에 진입하면 | "주문 내역이 없습니다" 안내 메시지가 표시된다 |

## 관리자용 스토리 (Admin Stories)

### US-A01: 매장 인증

**스토리**: 매장 관리자로서, 매장 관리 시스템에 안전하게 로그인하고 싶다, 매장 운영을 관리하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 관리자 로그인 화면에서 | 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면 | 인증 성공 시 대시보드로 이동한다 |
| 2 | 로그인에 성공했을 때 | 세션이 생성되면 | JWT 토큰이 발급되고 16시간 동안 유효하다 |
| 3 | 로그인된 상태에서 | 브라우저를 새로고침하면 | 세션이 유지되어 재로그인 없이 사용 가능하다 |
| 4 | 16시간이 경과했을 때 | 다음 요청을 보내면 | 자동 로그아웃되고 로그인 화면으로 이동한다 |
| 5 | 잘못된 인증 정보로 | 반복 로그인을 시도하면 | 일정 횟수 초과 시 로그인 시도가 제한된다 |
| 6 | 비밀번호가 저장될 때 | 시스템이 처리하면 | bcrypt로 해싱되어 저장된다 |

---

### US-A02: 실시간 주문 모니터링

**스토리**: 매장 관리자로서, 들어오는 주문을 실시간으로 확인하고 상태를 관리하고 싶다, 빠르게 주문을 처리하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 대시보드에 접속한 상태에서 | 화면을 확인하면 | 테이블별 카드가 그리드 레이아웃으로 표시된다 |
| 2 | 테이블 카드가 표시된 상태에서 | 각 카드를 확인하면 | 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다 |
| 3 | 고객이 새 주문을 생성했을 때 | 관리자 대시보드에서 | 2초 이내에 SSE를 통해 신규 주문이 표시된다 |
| 4 | 신규 주문이 도착했을 때 | 대시보드에서 확인하면 | 색상 변경/애니메이션으로 시각적 강조가 된다 |
| 5 | 테이블 카드를 | 클릭하면 | 해당 테이블의 전체 주문 메뉴 목록 상세가 표시된다 |
| 6 | 주문 상세에서 | 상태 변경 버튼을 누르면 | 대기중 → 준비중 → 완료 순서로 상태가 변경된다 |
| 7 | 대시보드에서 | 테이블 필터를 적용하면 | 선택한 테이블의 주문만 표시된다 |

---

### US-A03-1: 테이블 초기 설정

**스토리**: 매장 관리자로서, 태블릿에 테이블 정보를 설정하고 싶다, 고객이 해당 테이블에서 주문할 수 있도록 하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 테이블 관리 화면에서 | 테이블 번호와 비밀번호를 설정하면 | 테이블 정보가 저장되고 16시간 세션이 생성된다 |
| 2 | 설정이 완료되었을 때 | 태블릿에서 해당 정보로 로그인하면 | 자동 로그인이 활성화된다 |
| 3 | 이미 설정된 테이블에 | 동일한 번호로 재설정하면 | 기존 설정이 업데이트된다 |

---

### US-A03-2: 주문 삭제

**스토리**: 매장 관리자로서, 잘못된 주문을 삭제하고 싶다, 정확한 주문 현황을 유지하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 주문 상세 화면에서 | 주문 삭제 버튼을 누르면 | 확인 팝업이 표시된다 |
| 2 | 확인 팝업에서 | 삭제를 확정하면 | 주문이 즉시 삭제된다 |
| 3 | 주문이 삭제된 후 | 테이블 카드를 확인하면 | 총 주문액이 재계산되어 반영된다 |
| 4 | 삭제가 완료되었을 때 | 화면에서 | 성공 피드백 메시지가 표시된다 |
| 5 | 삭제가 실패했을 때 | 화면에서 | 실패 피드백 메시지가 표시되고 주문은 유지된다 |

---

### US-A03-3: 테이블 세션 처리 (이용 완료)

**스토리**: 매장 관리자로서, 고객이 떠난 후 테이블을 리셋하고 싶다, 새 고객이 깨끗한 상태에서 시작할 수 있도록 하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 테이블에 활성 세션이 있을 때 | 이용 완료 버튼을 누르면 | 확인 팝업이 표시된다 |
| 2 | 확인 팝업에서 | 이용 완료를 확정하면 | 현재 세션의 주문 내역이 과거 이력(OrderHistory)으로 이동한다 |
| 3 | 이용 완료 처리 후 | 테이블 카드를 확인하면 | 현재 주문 목록이 비어있고 총 주문액이 0원이다 |
| 4 | 이용 완료 처리 후 | 새 고객이 주문을 시작하면 | 이전 주문 내역 없이 새 세션이 시작된다 |
| 5 | 이용 완료가 실패했을 때 | 화면에서 | 실패 피드백 메시지가 표시되고 세션은 유지된다 |

---

### US-A03-4: 과거 주문 내역 조회

**스토리**: 매장 관리자로서, 테이블의 과거 주문 이력을 조회하고 싶다, 매장 운영 현황을 파악하기 위해.

**우선순위**: Should

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 테이블 관리 화면에서 | 과거 내역 버튼을 누르면 | 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다 |
| 2 | 과거 주문 목록에서 | 각 주문을 확인하면 | 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다 |
| 3 | 과거 내역 화면에서 | 날짜 필터를 적용하면 | 선택한 날짜 범위의 주문만 표시된다 |
| 4 | 과거 내역 화면에서 | 닫기 버튼을 누르면 | 대시보드로 복귀한다 |

---

### US-A04: 메뉴 관리

**스토리**: 매장 관리자로서, 메뉴를 등록/수정/삭제하고 싶다, 최신 메뉴 정보를 고객에게 제공하기 위해.

**우선순위**: Must

**수용 기준**:

| # | Given | When | Then |
|---|---|---|---|
| 1 | 메뉴 관리 화면에서 | 카테고리를 선택하면 | 해당 카테고리의 메뉴 목록이 표시된다 |
| 2 | 메뉴 등록 화면에서 | 메뉴명, 가격, 설명, 카테고리, 이미지 URL을 입력하고 저장하면 | 새 메뉴가 등록된다 |
| 3 | 메뉴 등록 시 | 필수 필드(메뉴명, 가격, 카테고리)가 비어있으면 | 검증 에러 메시지가 표시된다 |
| 4 | 메뉴 등록 시 | 가격이 유효 범위를 벗어나면 | 검증 에러 메시지가 표시된다 |
| 5 | 기존 메뉴를 | 수정하고 저장하면 | 변경 사항이 즉시 반영되고 고객 화면에도 적용된다 |
| 6 | 메뉴를 | 삭제하면 | 해당 메뉴가 목록에서 제거되고 고객 화면에서도 사라진다 |
| 7 | 메뉴 목록에서 | 노출 순서를 조정하면 | 고객 화면에서 변경된 순서로 표시된다 |

## INVEST 기준 검증

| 기준 | 설명 | 충족 여부 |
|---|---|---|
| **I**ndependent | 각 스토리가 독립적으로 구현/테스트 가능 | O - 스토리 간 의존성 최소화 |
| **N**egotiable | 구현 방식에 대해 협상 가능 | O - "무엇"을 정의하고 "어떻게"는 열어둠 |
| **V**aluable | 각 스토리가 사용자에게 가치 제공 | O - 모든 스토리가 사용자 목표와 연결 |
| **E**stimable | 구현 규모 추정 가능 | O - 수용 기준으로 범위 명확 |
| **S**mall | 하나의 반복(iteration)에서 구현 가능한 크기 | O - FR-A03을 4개 하위 스토리로 분리 |
| **T**estable | 수용 기준으로 테스트 가능 | O - Given/When/Then 형식으로 테스트 케이스 도출 가능 |
