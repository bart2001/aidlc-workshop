# 테이블오더 서비스 - 사용자 페르소나

## Persona 1: 매장 고객 (Customer)

| 항목 | 내용 |
|---|---|
| **이름** | 김민수 (대표 페르소나) |
| **역할** | 매장에서 식사하는 고객 |
| **연령대** | 20~50대 (폭넓은 연령층) |
| **기술 수준** | 초~중급 - 스마트폰/태블릿 기본 조작 가능 |
| **사용 환경** | 매장 테이블에 설치된 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 대기 시간 없이 빠르게 음식을 주문하고 싶다
- 메뉴를 쉽게 탐색하고 원하는 음식을 찾고 싶다
- 주문 상태를 실시간으로 확인하고 싶다

### 동기 (Motivations)
- 직원을 부르지 않고 편하게 주문하고 싶다
- 주문 실수를 줄이고 싶다 (직접 확인 후 주문)
- 추가 주문을 부담 없이 하고 싶다

### 불만 사항 (Pain Points)
- 직원을 기다리는 시간이 길다
- 메뉴판이 복잡하거나 가격이 잘 안 보인다
- 주문이 제대로 들어갔는지 불안하다

### 사용 시나리오
1. 테이블에 앉으면 태블릿이 이미 켜져 있고 메뉴 화면이 보인다
2. 카테고리별로 메뉴를 탐색하고 장바구니에 담는다
3. 주문을 확정하고 주문 번호를 확인한다
4. 추가 주문이 필요하면 같은 과정을 반복한다
5. 주문 내역에서 현재까지의 주문과 상태를 확인한다

## Persona 2: 매장 관리자 (Admin)

| 항목 | 내용 |
|---|---|
| **이름** | 박서연 (대표 페르소나) |
| **역할** | 매장 운영 관리자 (사장 또는 매니저) |
| **연령대** | 30~50대 |
| **기술 수준** | 중급 - 웹 서비스 사용에 익숙, 기본적인 IT 지식 보유 |
| **사용 환경** | 매장 내 PC 또는 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 들어오는 주문을 실시간으로 파악하고 빠르게 처리하고 싶다
- 메뉴를 쉽게 등록/수정/삭제하고 싶다
- 테이블별 주문 현황을 한눈에 보고 싶다
- 매장 이용 완료 후 테이블을 깔끔하게 리셋하고 싶다

### 동기 (Motivations)
- 주문 누락을 방지하고 싶다
- 인건비를 절감하고 운영 효율을 높이고 싶다
- 매장 상황을 실시간으로 파악하고 싶다

### 불만 사항 (Pain Points)
- 바쁜 시간대에 주문이 밀리면 놓치기 쉽다
- 메뉴 변경 시 종이 메뉴판을 다시 만들어야 한다
- 테이블별 주문 금액을 수동으로 계산해야 한다

### 사용 시나리오
1. 매장 오픈 시 관리자 화면에 로그인한다
2. 대시보드에서 테이블별 주문 현황을 실시간으로 모니터링한다
3. 신규 주문이 들어오면 상태를 "준비중"으로 변경한다
4. 음식이 나가면 "완료"로 변경한다
5. 고객이 떠나면 "이용 완료" 처리로 테이블을 리셋한다
6. 필요 시 메뉴를 등록/수정/삭제한다
