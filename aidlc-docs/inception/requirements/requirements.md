## 3. 아키텍처 결정사항

| 결정 항목 | 선택 | 근거 |
|---|---|---|
| **멀티테넌트** | 멀티테넌트 구조 | 하나의 시스템에서 여러 매장 지원, store_id로 데이터 격리 |
| **프론트엔드 구조** | 단일 프로젝트, 라우팅 분리 | 고객용/관리자용을 하나의 React 프로젝트에서 관리 |
| **관리자 계정 생성** | 시드(seed) 데이터 | MVP 단계에서 DB 시드 스크립트로 매장+관리자 생성 |
| **메뉴 이미지** | 외부 URL 직접 입력 | 이미지 업로드 기능 없이 URL만 저장 |
| **고객 실시간 업데이트** | SSE 포함 | 고객 화면에서도 주문 상태 실시간 업데이트 |

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 태블릿 초기 설정 시 매장 식별자, 테이블 번호, 테이블 비밀번호를 입력
- 테이블 비밀번호는 태블릿-테이블 매핑 확인용 인증 수단
- 1회 로그인 성공 후 로컬 저장된 정보로 자동 로그인
- 16시간 세션 유지 (JWT 토큰 기반)

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면으로 항상 표시
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 → 5초 후 장바구니 자동 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 주문 메뉴 목록(메뉴명, 수량, 단가), 총 주문 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- SSE를 통한 주문 상태 실시간 업데이트
- 매장 이용 완료 처리된 주문 제외

### 4.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱 저장
- 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **태블릿 초기 설정**: 테이블 번호/비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 주문 즉시 삭제 → 테이블 총 주문액 재계산
- **테이블 세션 처리**:
  - 세션 시작: 해당 세션의 첫 주문 시 자동 시작
  - 세션 종료(이용 완료): 확인 팝업 → 주문 내역을 과거 이력으로 이동 → 현재 주문 목록/총 주문액 리셋
  - 새 고객이 이전 주문 내역 없이 시작 가능
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록(시간 역순), 날짜 필터링, 닫기 버튼으로 대시보드 복귀

#### FR-A04: 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 정보: 메뉴명, 가격, 설명, 카테고리, 이미지 URL
- 카테고리별 조회
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 5. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE 기반 실시간 통신: 주문 발생 후 2초 이내 관리자/고객 화면 반영
- 메뉴 조회 API 응답 시간: 500ms 이내
- 동시 접속: 매장당 최소 30개 테이블 + 관리자 동시 지원

### NFR-02: 보안
- JWT 토큰 기반 인증 (16시간 만료)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한 (브루트포스 방지)
- HTTPS 통신 필수
- 멀티테넌트 데이터 격리 (store_id 기반)

### NFR-03: 사용성
- 터치 친화적 UI (최소 44x44px 터치 타겟)
- 카드 형태 메뉴 레이아웃
- 명확한 시각적 계층 구조
- 직관적인 관리자 도구

### NFR-04: 신뢰성
- 장바구니 로컬 저장 (페이지 새로고침 시 유지)
- 주문 실패 시 장바구니 유지
- SSE 연결 끊김 시 자동 재연결

### NFR-05: 데이터 관리
- 주문 이력 데이터베이스 영구 저장 (OrderHistory)
- 세션 ID로 주문 그룹화
- 완료 시각 기록

## 6. 제외 범위 (Out of Scope)

다음 기능은 MVP에서 구현하지 않습니다:

- **결제**: 카드/현금/디지털 지갑 결제, PG사 연동, 영수증, 환불, 포인트/쿠폰
- **고급 인증**: OAuth, SNS 로그인, 2FA/OTP
- **파일 관리**: 이미지 업로드/리사이징, CMS, 광고
- **알림**: 푸시, SMS, 이메일, 소리/진동
- **주방**: 주방 전달, 재고 관리
- **분석**: 대시보드, 매출 리포트, 재고 관리, 직원 관리, 예약, 리뷰, 다국어
- **외부 연동**: 배달 플랫폼, POS, 소셜 미디어, 지도/번역 API

## 7. 데이터 모델 개요

### 핵심 엔티티
- **Store**: 매장 정보 (멀티테넌트 기준)
- **Admin**: 매장 관리자 계정
- **Table**: 테이블 정보 (매장별)
- **TableSession**: 테이블 세션 (고객 이용 단위)
- **Category**: 메뉴 카테고리
- **MenuItem**: 메뉴 항목
- **Order**: 주문 정보
- **OrderItem**: 주문 상세 항목
- **OrderHistory**: 과거 주문 이력 (세션 종료 후 이동)

### 멀티테넌트 데이터 격리
- 모든 엔티티에 `store_id` 외래키 포함
- API 레벨에서 store_id 기반 데이터 필터링 적용

## 8. 사용자 유형

| 사용자 | 설명 | 인증 방식 |
|---|---|---|
| **고객** | 테이블에서 메뉴를 주문하는 사람 | 태블릿 자동 로그인 (테이블 비밀번호 기반) |
| **매장 관리자** | 매장 운영을 관리하는 사람 | 매장ID + 사용자명 + 비밀번호 |
