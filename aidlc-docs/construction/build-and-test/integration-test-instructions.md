# 테이블오더 서비스 - Integration Test Instructions

## 1. Frontend ↔ Backend API 연동 테스트

### 1.1 테스트 환경

```bash
# Backend 실행 (test 프로파일)
cd backend
./gradlew bootRun --args='--spring.profiles.active=test'

# Frontend 실행 (API 연동 모드)
cd frontend
VITE_API_URL=http://localhost:8080 VITE_USE_MOCK=false npm run dev
```

### 1.2 API 연동 테스트 항목

| # | API | 테스트 내용 | 검증 항목 |
|---|---|---|---|
| 1 | POST /api/auth/admin/login | 관리자 로그인 | JWT 토큰 반환, 상태 200 |
| 2 | GET /api/stores/{id}/categories | 카테고리 목록 조회 | 인증 헤더 전송, 데이터 반환 |
| 3 | POST /api/stores/{id}/categories | 카테고리 등록 | 요청 바디 형식, 201 응답 |
| 4 | GET /api/stores/{id}/menus | 메뉴 목록 조회 | 카테고리 필터 파라미터 |
| 5 | POST /api/stores/{id}/menus | 메뉴 등록 | 필수 필드 검증, 가격 범위 |
| 6 | GET /api/stores/{id}/orders | 주문 목록 조회 | 관리자 권한 확인 |
| 7 | PATCH /api/stores/{id}/orders/{id} | 주문 상태 변경 | 상태 전이 규칙 |
| 8 | DELETE /api/stores/{id}/orders/{id} | 주문 삭제 | 삭제 후 목록 반영 |
| 9 | GET /api/stores/{id}/tables | 테이블 목록 조회 | 관리자 권한 확인 |
| 10 | PATCH /api/stores/{id}/tables/{id}/sessions/{id} | 이용 완료 | 세션 종료, 이력 이동 |

### 1.3 CORS 검증

- Frontend (localhost:5173) → Backend (localhost:8080) 간 CORS 헤더 확인
- Preflight OPTIONS 요청 정상 응답 확인
- Authorization 헤더 허용 확인

## 2. SSE 실시간 통신 테스트

### 2.1 관리자용 SSE 스트림

| # | 시나리오 | 테스트 방법 | 기대 결과 |
|---|---|---|---|
| 1 | SSE 연결 수립 | 대시보드 접속 후 DevTools Network 탭 확인 | EventStream 연결 활성 |
| 2 | 신규 주문 수신 | 고객 화면에서 주문 생성 | 2초 이내 대시보드에 신규 주문 표시 |
| 3 | 주문 상태 변경 수신 | 다른 관리자 세션에서 상태 변경 | 실시간 상태 업데이트 |
| 4 | 주문 삭제 수신 | 주문 삭제 실행 | 대시보드에서 즉시 제거 |
| 5 | 세션 종료 수신 | 이용 완료 처리 | 해당 테이블 주문 목록 초기화 |
| 6 | 연결 끊김 복구 | 네트워크 일시 차단 후 복구 | 3초 후 자동 재연결 |
| 7 | 시각적 강조 | 신규 주문 도착 | 테이블 카드 색상 변경 (5초간) |

### 2.2 SSE 테스트 스크립트

```bash
# curl로 SSE 스트림 직접 테스트
TOKEN="<JWT_TOKEN>"
curl -N -H "Accept: text/event-stream" \
  -H "Authorization: Bearer $TOKEN" \
  http://localhost:8080/api/stores/1/orders/stream
```

| # | 시나리오 | 기대 결과 |
